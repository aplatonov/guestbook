Плюсы:
1. Реализовано большое количество функционала
2. Реализованы функции для экранирования и валидации данных перед передачей их в БД

Ошибки / замечания:
1. Нет отделения логики от представления.
2. Для защиты подключаемых файлов корректнее было бы использовать константу, которая бы объявлялась в входных файлах
3. Редиректы корректнее делать на PHP с использованием функции header()
4. Не везде есть проекврка корректности входных данных перед использованием их в SQL запросах. (sql injection)
5. Нет проверки на существование данных для вывода страницы. Пример: http://test17.learning.dev1.msoft.su/index.php?page=10000


Рекомендации:
1. Необходимо изучить и придерживаться стандартам оформления кода PSR-1, PSR-2.
2. Изучить формат написания комментариев PHPDoc.
3. Для отделения логики от представления рекомендую использовать шаблонизаторы. Наиболее популярные Twig, Smarty
